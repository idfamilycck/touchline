# 더그아웃 (DUGOUT) — 당신이 감독이라면: 설계 문서

- 날짜: 2026-07-17
- 대상: 데이콘 월간 해커톤 "내가 축구 감독이라면 — 월드컵 전술 웹서비스 챌린지"
- 참가 형태: 개인, Claude Code 풀활용
- 최종 제출 마감: 2026-08-03(월) 10:00 (기획서 PDF: 2026-07-27(월) 10:00)

## 1. 한 줄 정의

상대팀·경기장을 고르고 선수를 드래그로 배치하면, 통계 엔진이 **조작할 때마다 실시간으로 승률을 갱신**하고 근거와 함께 최적 전술을 추천하며, 그 전술로 경기를 시뮬레이션해 **감독 개입이 결과를 어떻게 바꿨는지 수치로 복기**해주는 웹서비스.

## 2. 우승 전략 → 기능 매핑

| 평가 항목 (배점) | 대응 장치 |
|---|---|
| 참신성 (30) | ① 모든 조작에 실시간 반응하는 승률 ② 2026 실제 개최지 환경 변수(고도·더위·돔)가 전술 유불리를 바꿈 ③ "개입이 없었다면" 카운터팩추얼 복기 — 같은 시드로 무개입 경기를 재시뮬해 내 결정의 가치를 수치화 |
| 감독 경험 설계 (25) | **3계층 전술 시스템(팀 지시 → 선수 역할 → 맨마킹·세트피스 특수 지시)**, 드래그 배치, 자유 일시정지 지시, 위기 순간 알림, 방송 중계형 연출, 승부차기 키커 순서 지정, 퀵스타트·온보딩 코치마크 |
| 완성도 (25) | 100% 클라이언트 엔진(외부 API·키 없음 → 심사 중 장애 원천 차단), 시드 기반 재현 가능한 시뮬레이션, 엔진 단위 테스트 |
| 기획/구현 완성도 (20) | 기획서의 인터랙션 명세 = 실제 화면 1:1 대응, README에 기획→구현 매핑표 수록 |

1차 대중 투표(제출팀 상호 60%) 대비: 첫인상 비주얼(방송 그래픽 스타일), OG 공유 카드, 시연 영상 스토리("0-1 뒤지던 경기를 교체로 뒤집는" 서사).

AI 채점 가능성 대비: 시맨틱 마크업·aria 라벨, 콘솔 에러 제로, README에 엔진 수식·실행 방법·기획 매핑 완비, 기획서와 서비스 용어 통일.

## 4. 사용자 흐름 & 화면 명세 (4 + 1 화면)

### 4.1 홈 — 매치업 설정
- **퀵스타트(최우선 요소)**: "한국 vs 브라질 바로 지휘하기" 등 추천 매치업 원클릭 버튼 → 프리셋 라인업·기본 전술로 즉시 작전실 진입. 첫 방문자가 10초 안에 승률이 반응하는 핵심 경험에 도달하는 것이 목표(대중투표 대응)
- 16개국 카드 그리드에서 내 팀 → 상대 팀 순서로 선택 (자체 제작 국기 배지, 저작권 안전)
- 경기장 선택: 2026 실제 개최지 기반 경기장 목록(고도·기온·돔 여부 아이콘 표시). 선택이 전술 유불리에 영향을 준다는 안내
- 팀 카드에 ELO·최근 폼·상대전적 요약 노출
- 푸터 고지: 모든 선수·팀 능력치는 가상으로 구성한 데이터임을 명시(공정성·명예훼손 리스크 차단). 선수 사진·초상은 사용하지 않고 이니셜 아바타로 대체

### 4.2 작전실 — 전술 보드 (핵심 화면)
- 좌: 스쿼드 리스트(포지션·능력치·나이·컨디션), 중앙: 2D 피치 드래그 배치(dnd-kit), 우: 승률 게이지 + 라인별 전력 비교 + 추천 전술 패널
- **전술 시스템 3계층 — "감독의 의도"를 세부까지 반영**:
  - **① 팀 지시**: 포메이션 6종(4-3-3, 4-4-2, 4-2-3-1, 3-5-2, 3-4-3, 5-4-1) + 압박 강도·수비 라인 높이·공격 성향·템포 슬라이더 + 빌드업 스타일(후방 숏패스/다이렉트 롱볼) + 공격 전개 방향(좌측면/중앙/우측면) + 공격 폭(넓게/좁게) + 수비 방식(지역/대인) + 오프사이드 트랩 on/off
  - **② 선수 역할(Role)**: 같은 포지션이라도 역할 선택 — ST: 타겟맨/폴스나인/포처, 윙어: 인버티드/클래식, 중앙 MF: 딥라잉 플레이메이커/박스투박스/홀딩, 풀백: 오버래핑/수비형 등 (포지션당 2~3종). 역할별로 반영되는 능력치 가중이 달라짐(타겟맨=공중볼·피지컬, 폴스나인=패스·드리블)
  - **③ 특수 지시**: **맨마킹 지정**(우리 선수 1명이 상대 에이스 전담 — 피치 위 연결선으로 시각화, 상대 위협도 감소 vs 마커의 공격 기여·체력 소모 트레이드오프), 세트피스 설계(코너·프리킥 키커 지정, 코너 시 장신 수비수 전진 여부), 주장 지정(멘탈 보정)
  - 특수 지시는 피치 위에서 항상 식별 가능해야 함: 선수 아바타에 주장 `C`·프리킥 `FK`·코너 `CK` 미니 배지, 맨마킹은 대상 연결선 — 설정이 실제로 작동함을 시각으로 증명
- 모든 계층의 선택이 승률·근거 카드에 반영됨 (예: "다이렉트 롱볼 + 타겟맨 조합 → 상대 높은 라인 배후 공략 +6%p", "맨마킹: 음바페 위협도 -31%, 김민재 공격 가담 -8%")
- **모든 조작(선수 이동·교체, 포메이션·슬라이더 변경)이 승률을 즉시 갱신** — 변동량(+2.4%p 등)을 애니메이션으로 표시
- "추천 전술 보기" → 최고 승률 조합 제시 + 근거 분해 카드("상대 측면 수비 71 vs 우리 윙 스피드 88 → 측면 공격 +9%p", "아스테카 고도 → 높은 압박 체력 페널티 -4%p")
- 포지션 부적합 배치(윙어를 CB에) 시각 경고
- 키보드 대체 수단: 선수 선택 → 슬롯 선택으로 배치 가능(접근성)
- **온보딩**: 첫 진입 시 "감독 브리핑" 콘셉트의 3단계 코치마크(① 선수를 끌어 배치 ② 승률 변동 확인 ③ 경기 시작). localStorage로 1회만 표시, 건너뛰기 가능

### 4.3 경기 — 시뮬레이션
- 상단: 2D 피치에서 주요 장면(공격 전개·슈팅·골·코너·카드)을 점(선수)·공 이동 애니메이션으로 재생, 스코어보드·경기 시간
- 하단: 텍스트 중계 피드(캐스터 톤) + 실시간 승률 타임라인 그래프(개입 시점 마커)
- 자유 일시정지 → 교체(최대 5명)·포메이션·지시 변경 → 엔진 재계산 후 재개
- 위기 알림: 실점 직후·연속 위기·체력 급감 시 "감독님, 지시가 필요합니다" 배너로 개입 유도
- 후반 체력 감쇠로 교체가 실제 승률에 유의미하게 작용
- 배속 조절(1x/2x/4x), 시드 고정으로 동일 조건 재현 가능

### 4.4 승부차기 (조건부 화면)
- 시뮬 종료가 무승부이면 "무승부입니다. 승부차기로 결판내시겠습니까?" 선택지 제시(수락/무승부 종료)
- 사용자가 키커 5명과 순서를 드래그로 지정 (PK 능력·멘탈 스탯 노출), 상대는 엔진이 자동 배정
- 킥마다 1회 클릭 진행, PK 능력 vs GK 선방 확률 대결 연출

### 4.5 복기 — 리포트
- 최종 결과 + 승률 타임라인(개입 마커) + 주요 스탯
- **카운터팩추얼**: 같은 시드로 무개입(초기 전술 유지) 경기를 백그라운드 재시뮬 → "당신의 63' 교체가 승률을 +14%p 바꿨습니다" / 무개입 평행세계 스코어 비교
- 결과 공유 카드(클라이언트 이미지 저장·링크 복사). OG 메타태그는 정적(서비스 대표 카드) — 매치별 동적 OG는 서버가 필요하므로 비범위
- "다시 도전" → 같은 매치업 재도전 또는 홈으로

### 4.6 데이터 가시화 원칙 (모든 화면 공통) — "엔진은 복잡하게, 화면은 단순하게"

수많은 변수(능력치 8종 × 역할 × 상성 × 환경 × 히스토리)가 계산에 쓰이지만, 화면에는 **일반인이 3초 안에 읽는 형태**로만 노출한다.

- **정보 3계층 (Progressive Disclosure)**:
  - 1층(항상 표시): 승률 게이지 큰 숫자 하나 + 라인별 전력 막대 대결(우리 vs 상대) + 추천 한 줄
  - 2층(근거 카드): 요인당 "아이콘 + 자연어 한 문장 + 델타"만 — 예: "🏔 고지대라 강한 압박은 후반에 지칩니다 −4%p"
  - 3층(펼치기 선택 시): 상세 수치·산식 설명. 궁금한 사람과 심사자만 연다
- **3중 인코딩 통일**: 유리 = 초록·↑, 불리 = 빨강·↓, 항상 색+방향+수치를 함께 표기(색약 대응 포함). 승률 변동은 카운트업 애니메이션으로 "방금 바뀌었음"을 체감시킴
- **조작→반응 1:1 피드백**: 무언가를 바꾸면 그 조작 때문에 변한 근거 카드만 하이라이트 — "내 결정이 무엇을 바꿨는지" 즉시 연결
- **용어 규칙**: 화면 문구는 축구 팬이 아닌 사람 기준(예: "점유율 지향" 대신 "공을 오래 소유하며 풀어가기"), 전문 용어는 툴팁으로 보조
- **시연 최적화**: 핵심 숫자는 원거리·영상에서도 읽히는 큰 타이포와 고대비, 경기 화면의 승률 타임라인은 골·개입 마커 중심의 단순 라인 하나만
- 복기 화면도 동일 원칙: 그래프 하나 + 결론 한 문장("당신의 63' 교체가 승률을 +14%p 바꿨습니다")이 주인공, 상세 표는 접힘

## 5. 통계 엔진 명세 (`lib/engine/`, 순수 TS, UI 무의존)

### 5.1 데이터 (`data/*.json`, 더미 데이터 직접 구성 — 대회 권장 방식)
- **teams.json**: 16개국. 팀별 ELO 레이팅, FIFA 랭킹, 최근 10경기 폼 지수, 스타일 태그
- **players.json**: 팀당 ~20명. 이름(실제 선수 참고)·포지션(주/부)·나이·A매치 수·능력치 8종(슈팅·패스·드리블·수비·스피드·피지컬·GK·체력)·세트피스·공중볼·PK·멘탈
- **venues.json**: 2026 실제 개최지 기반 경기장. 고도(m)·평균 기온·돔 여부·수용 인원
- **h2h.json**: 국가 쌍별 역대 상대전적(승무패) — 심리 보정용
- **tactics.ts**: 포메이션 정의(슬롯 좌표·포지션 요구), 선수 역할 정의(역할별 능력치 가중 프로파일), 팀 지시·스타일 상성 매트릭스(빌드업·방향·폭·수비방식·트랩 포함)

### 5.2 산출 파이프라인
1. **선수 기여도**: 능력치 × 역할 가중 프로파일 × 포지션 적합도(부적합 배치 페널티) × 나이 커브(포지션별 피크 상이) × 컨디션
2. **라인별 전력**: 수비/중원/공격 3라인 집계 + GK. 맨마킹 지정 시 대상 위협도 감쇠·마커 기여 재배분
3. **전술 보정**: 팀 지시 전체(포메이션·슬라이더·빌드업·방향·폭·수비방식·트랩) → 스타일 상성 매트릭스 (예: 높은 라인 vs 상대 스피드형 역습 = 불리, 다이렉트 롱볼 × 타겟맨 = 시너지)
4. **환경 보정**: 고도 → 체력 감쇠 가속·높은 압박 페널티, 폭염(비돔) → 후반 전력 감쇠 가속
5. **히스토리 보정**: ELO 차이(베이스라인), 폼 지수, 상대전적(소폭 심리 보정, 상한 캡)
6. **기대득점 λ 산출** → 포아송 분포로 승/무/패 확률 계산
7. **추천 탐색**: 탐색 공간은 **팀 지시(포메이션 × 슬라이더·지시 이산화)로 한정**해 전수 평가(약 2.3만 조합의 순수 산술 — 저사양 기기에서도 수십 ms). **"추천 전술 보기" 클릭 시 온디맨드로만 실행**하며 조작마다 도는 것은 단일 승률 평가뿐. 실측 100ms 초과 시에만 Web Worker 분리를 폴백으로 검토(선제 도입 금지). 선수 배치·역할은 조합 폭발 방지를 위해 별도 단계 — 포지션·역할 적합도 기반 그리디 자동 최적 배치로 산출. 결과 = 최고 승률 조합 + 요인별 기여 분해(근거 카드 데이터). 맨마킹은 상대 최고 위협 선수 대상 시 승률 변화만 제안
8. **경기 시뮬**: 시드 기반(seedrandom) 분 단위 이벤트 체인 생성(찬스→슈팅→골/세이브, 코너, 카드), 분당 λ를 시간·체력·전술 변경에 따라 갱신
9. **승부차기**: 키커 PK·멘탈 vs GK 선방 확률 대결
10. **카운터팩추얼**: 동일 시드 + 개입 로그 무시 재시뮬 → 개입별 승률 델타 산출

### 5.3 품질 규칙
- 엔진은 순수 함수로만 구성(입력 → 출력, 부작용 없음), vitest 단위 테스트(승률 합=1, 극단 입력 안정성, 시드 재현성, 환경 보정 방향성, 역할·맨마킹 보정 방향성)
- **카운터팩추얼 정합성 불변식 테스트**(참신성 30점의 킬러 피처 방어): ① 무개입 재시뮬은 원 경기의 개입 이전 구간과 이벤트가 완전 동일(동일 시드) ② 원 경기의 개입 로그를 재적용하면 원 경기 100% 재현 ③ 개입 없는 경기의 카운터팩추얼 델타 = 정확히 0
- **밸런싱 검증(납득감 테스트)**: 전 매치업(16×15) 몬테카를로 시뮬 배치를 테스트로 자동화 — ELO 우위팀 승률이 상식 범위(예: 최강 vs 최약 70~90%)에 들고, 5골 차 이상 스코어 빈도가 임계치 이하인지 검증. 통계 서비스에서 "이상한 결과"는 버그보다 치명적이므로 완성도 25점의 핵심 방어선
- 승률·추천 계산은 동기 실행 100ms 이내 목표(팀 지시로 한정한 조합 공간 규모상 충분)

## 6. 기술 아키텍처

- **스택**: Next.js 15(App Router) + TypeScript + Tailwind CSS, 상태 Zustand, 드래그 dnd-kit, 애니메이션 Framer Motion + SVG, 차트 자체 SVG

- **폴더**: `app/` `components/` `lib/engine/` `lib/engine/__tests__/` `data/` `types/`
- **배포**: Vercel, **`output: 'export'` 완전 정적 빌드**(서버 기능 전무를 빌드 타임에 강제 — 서버 의존이 실수로 스며드는 것을 차단, `next/image`는 unoptimized 설정)
- **모바일 명세(대중투표자 다수가 모바일)**: 작전실은 데스크톱 3열 → 모바일에서 탭 전환(스쿼드/피치/분석) 레이아웃. dnd-kit 터치 센서 활성화 + 탭-투-배치(선수 탭 → 슬롯 탭) 대체 수단. 경기 화면은 세로 스택. 모든 화면 375px 폭 기준 검수
- **에러 처리**: 엔진 입력 검증(11명 미만 배치 시 경기 시작 차단·안내), ErrorBoundary로 화면 단위 격리, 잘못된 라우트 진입 시 홈 리다이렉트
- **테스트**: 엔진 vitest + 핵심 플로우 Playwright 스모크(홈→배치→시뮬→복기 1회 완주)
